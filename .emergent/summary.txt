<analysis>**original_problem_statement:**
The user requested a series of major features to build out a comprehensive labour and materials management system.
**PRODUCT REQUIREMENTS:**
1.  **Enhanced Vendor Management**:
    *   A new Vendors menu in the Planning department header.
    *   A full CRUD interface for vendors with fields for Basic Details, Account Details, Brands, Material Categories, Payment Cycle, and GST.
    *   Vendor categories should be a dynamic, searchable dropdown with an add new option.
    *   A vendor detail view with tabs for Payment Summary and Order Summary.
    *   In the project planning phase, the ability to assign specific vendors to material categories for that project.
2.  **Contractor (Labour) Management**:
    *   A new Contractors menu in the Planning header.
    *   A full CRUD interface for contractors, similar to vendors.
    *   Contractor types (e.g., Mason, Painter) should be a dynamic, manageable category list.
    *   For each contractor, define specific labour types (e.g., Skilled, Semi-Skilled) and their per-day costs.
    *   A contractor detail view with tabs for Work Orders and Payment Summary.
3.  **Work Order Management**:
    *   Within a project's detail page, a Work Orders tab where Planning can create work orders for assigned contractors.
    *   A work order should define the total value and be broken down into multiple payment stages (e.g., Stage 1 - 20%).
4.  **Daily Labour Attendance**:
    *   The Site Engineer should see assigned contractors for the project.
    *   A new Attendance tab where the Site Engineer can log the number of labourers (by skill type) who worked on a specific work order stage each day.
    *   The system should automatically calculate daily labour costs based on attendance.
5.  **Labour Payment Workflow**:
    *   The Site Engineer can request payment for a completed work order stage.
    *   The request goes to Planning for review. Planning can approve or modify the amount before forwarding it to the Accountant.
6.  **Material Inventory Management**:
    *   In the project's Materials tab, add an Inventory section.
    *   After materials are received, the Site Engineer must log daily opening and closing stock to maintain a historical inventory record.
